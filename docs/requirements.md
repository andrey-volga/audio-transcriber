# Требования к проекту: Audio Transcriber

## Общее описание

CLI-инструмент для локальной транскрибации аудио/видео-файлов в текст с использованием OpenAI Whisper. Работает полностью офлайн, без внешних API.

---

## Функциональные требования

### FR-01: Входные данные
- Поддерживаемые форматы: **MP3, WAV, MP4, M4A, OGG, FLAC**
- Источники:
  - Отдельный файл: `transcribe audio.mp3`
  - Папка (batch-режим): `transcribe ./recordings/` — обрабатывает все аудио-файлы в директории

### FR-02: Транскрибация
- Движок: **OpenAI Whisper (локальный)**
- Среда выполнения: **CPU** (без GPU)
- Язык транскрибации: **Русский** (явно задан, без авто-определения)

### FR-03: Выходные данные
- Формат: **TXT**
- Содержимое: чистый текст транскрипции без таймкодов
- Имя выходного файла: совпадает с именем входного (например, `audio.mp3` → `audio.txt`)

### FR-04: CLI-интерфейс
- Единственный интерфейс взаимодействия с системой
- Команды:
  ```
  transcribe <file>           # транскрибировать один файл
  transcribe <directory>      # транскрибировать все файлы в папке
  transcribe --help           # справка
  ```

---

## Нефункциональные требования

### NFR-01: Среда запуска
- Платформа: Linux
- Python 3.11+
- Работа без GPU (только CPU)
- Не требует интернет-соединения во время работы

### NFR-02: Производительность
- Обработка 1 минуты аудио: не более 5 минут на CPU (whisper-base)

### NFR-03: Надёжность
- При ошибке обработки одного файла в batch-режиме — продолжать обработку остальных
- Понятные сообщения об ошибках (неверный формат, файл не найден и т.п.)

---

## Ограничения MVP

- Нет REST API
- Нет веб-интерфейса
- Нет диаризации спикеров
- Нет субтитров (SRT)
- Нет таймкодов в выводе
- Нет поддержки URL/YouTube
